About WURM

Beyond make website

WURM (Web Upload with Recursive Makefiles) is a complete website management package in the form of a set of makefiles -- control files for GNU make. The general idea is that you can go into a web directory, edit a file, and give commands like:

  • make blog
  • make feeds
  • make upload
  • make backups

Similarly, you can plug in your digital camera and say:

  • make photos
  • make thumbnails
  • make gallery
  • make upload

(Actually, it's simpler than that because, unless you need to be selective for some reason, you can simply say make and everything that needs doing will get done: make is good that way.)

WURM also happily takes "plug-ins" in the form of make "rules" that tell it how to handle particular classes of files. If you want, for example, to run all your .tex files through latex to turn them into .pdf files, you can do it with about four lines of code. WURM will, in fact, work will with any program that can be run in batch mode from a command line, including such blogging programs as blosxom and nanoblogger, and even text editors like Emacs and image editors like the Gimp.

WURM works with multiple upload methods including ftp and WebDAV as well as the preferred rsync and ssh. The general idea is to do as much work as possible on your local system (or a local server) before anything gets uploaded -- in most cases, all your web server has to do is serve static files and perhaps handle the occasional comment from a reader.
